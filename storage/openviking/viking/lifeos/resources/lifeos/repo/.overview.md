# workspace

This workspace serves as the root directory for the **LifeOS** project, a comprehensive system designed to manage personal life, spirituality, and productivity workflows. It functions as a monorepo containing a multi-service application architecture, including a FastAPI backend, a web user interface, a Discord bot integration, and specialized AI workers for text-to-speech (TTS) and embedding capabilities. The directory covers the full stack of the application, from infrastructure orchestration to frontend visualization. Key technical components include GPU-enabled processing, microservices communication, and persistent storage management. This workspace is suitable for full-stack developers, DevOps engineers, and project contributors looking to deploy, extend, or maintain the LifeOS ecosystem.

## Detailed Description

### docker-compose.yml docker-compose.yml
This Docker Compose configuration file orchestrates a multi-service application deployment for the LifeOS project. It defines five distinct services: a GPU-enabled TTS worker for text-to-speech processing, an OpenViking service for embedding and VLM capabilities, a backend API service, a Discord bot integration, and a web user interface. The configuration establishes service dependencies with health checks ensuring proper startup order, implements networking through a custom bridge network, and manages volume mounts for persistent storage and configuration. Key technical features include NVIDIA GPU support, environment variable interpolation, and health monitoring endpoints. The architecture follows a microservices pattern where the backend depends on TTS and OpenViking services, while the Discord bot and web UI depend on the backend, creating a layered dependency chain with robust fault tolerance through restart policies and health validation mechanisms.
